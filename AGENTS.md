# Project rules

## Project

This project is a responsive booking administration dashboard built with:

- HTML
- CSS
- vanilla JavaScript

Do not use React, TypeScript, frameworks, build tools, npm packages or external UI libraries.

## File structure

Keep the project separated into:

- index.html
- style.css
- script.js

Do not place CSS or JavaScript inside index.html.

## Code quality

- Use semantic HTML.
- Use clear class names.
- Keep JavaScript functions small and focused.
- Store application data in arrays of objects.
- Render repeated interface elements from JavaScript.
- Do not duplicate HTML rows manually.
- Preserve accessibility attributes.
- Avoid unnecessary abstractions.

## Design

The dashboard should look like a practical business tool, not an AI-generated SaaS template.

Avoid:

- excessive gradients;
- glassmorphism;
- neon colors;
- huge rounded cards;
- excessive shadows;
- floating decorative objects;
- generic marketing text;
- unnecessary animations.

Use:

- restrained colors;
- compact spacing;
- clear hierarchy;
- readable tables;
- visible statuses;
- practical controls;
- subtle hover states.

## Token-efficient work mode

- Work only inside this project.
- Read only files related to the task.
- Make targeted edits.
- Do not rewrite entire files for a small correction.
- Do not inspect Git history unless requested.
- Do not update README unless requested.
- Do not add dependencies or additional files unless requested.
- Complete the task in one implementation pass.
- Run one final verification pass.
- Final response must contain no more than 8 short lines.
