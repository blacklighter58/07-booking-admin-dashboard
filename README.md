# Booking Desk — административная панель

Административный интерфейс системы онлайн-записи. Панель помогает сотруднику видеть актуальные записи, управлять клиентами, услугами и расписанием через Booking API.

## Бизнес-задача

Свести обработку записей в одно рабочее место: администратор авторизуется, просматривает календарь и заявки, меняет их статусы и поддерживает клиентскую базу без ручной сверки разных каналов.

## Возможности

- вход администратора по JWT;
- список заявок с поиском, фильтрами и сортировкой;
- обзор ближайших записей и показателей;
- календарь сотрудников;
- управление услугами, сотрудниками, клиентами и настройками;
- обновление данных из Booking API без кеширования устаревшего списка.

## Роль в системе

Панель — клиентская часть для сотрудников. Она обращается к `09-booking-api` по HTTPS и не хранит ключи API, пароли или строку подключения к базе данных.

```text
Администратор → Dashboard → Booking API → PostgreSQL / Neon
Telegram-бот ────────────────────────┘
```

## Технологии

- JavaScript и Vite;
- HTML/CSS;
- REST API и JWT;
- Vercel для публикации интерфейса.

## Локальный запуск

```powershell
Copy-Item .env.example .env.local
npm.cmd install
npm.cmd run dev
```

Настройте `VITE_API_URL` в соответствии с [.env.example](.env.example). Значение должно быть публичным HTTPS-адресом Booking API в production.

## Команды

```powershell
npm.cmd run check
npm.cmd test
npm.cmd run build
npm.cmd run preview
```

## Безопасность

- токен авторизации хранится только в сессии браузера;
- секреты не добавляются в клиентскую сборку;
- `.env*`, сборка и логи исключены из Git.

## Структура

test/         автоматические проверки
